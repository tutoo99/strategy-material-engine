## 三句话开发法

```
1. 截图 + "参考这个帮我做一个微信小程序"
   → AI生成完整代码框架

2. 截图 + "帮我调整成这样" + 效果图
   → AI修改样式

3. 截图 + "这里有问题继续修复"
   → AI修bug
```

## 常见坑

| 坑 | 现象 | 解决 |
|----|------|------|
| 权限没配置 | 模拟器OK真机闪退 | app.json声明权限 |
| 模拟器vs真机 | 布局全乱 | 必须手机扫码测试 |
| 审核被拒 | 功能简单/虚拟支付/内容敏感 | 按反馈改，2-3次过 |

## 核心心态

先上线再优化。你觉得完美的功能用户可能不在意，你觉得无所谓的细节用户可能天天在骂。只有上线了才知道真相。
