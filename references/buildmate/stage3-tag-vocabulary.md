# 阶段三最小词表（strategy_tags / resource_refs）

这份词表服务当前阶段三正式系统。

目标只有两个：

1. 避免相同策略在不同案例里用不同名字，导致无法稳定连边。
2. 先建立一版足够小、但已正式可用的标准词，而不是一开始追求覆盖一切。

## 使用规则

- `strategy_tags` 只写“策略动作”或“打法模块”，不要写太抽象的大词。
- `resource_refs` 只写“可调用的工具 / 平台 / 数据源 / 资源入口”。
- 一个案例优先控制在：
  - `strategy_tags`: `3~6` 个
  - `resource_refs`: `3~8` 个
- 能复用已有词就不要新造词。
- 如果确实需要新增词，优先满足：
  - 能直接连到节点
  - 能跨至少 2 个案例复用
  - 不是一次性描述
