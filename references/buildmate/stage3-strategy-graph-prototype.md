## 资源调用索引

本轮先把 `AP/TR/TC` 定义成“手工原型资源索引”，全部在本文件里闭环，不依赖新脚本。

### AP001 - 强结果选题拆解 SOP

适用目标：冷启动阶段获取第一批精准流量。

执行步骤：

1. 从自己的真实经历、结果或已验证案例里，挑一个能写成“身份 + 结果 + 时间/变化”的素材。
2. 把标题写成“结果承诺 + 特定人群 + 限定场景”的结构。
3. 正文或视频前 30 秒只做两件事：说明自己是谁、说明结果为什么值得听。
4. CTA 只保留一个，不同时让用户关注公众号、加微信、进群、私信四选一。
5. 发出后记录哪类评论和私信最多，作为下一轮内容的人群筛选信号。

案例来源：

- Luke 林案例中的“普通人如何七天赚到一万元”
- `bilibili_case_approved` 中“南方人去东北旅游怎么挑羽绒服”

### AP002 - 种子用户收费验证 SOP

适用目标：验证需求是否真实存在，验证用户是否愿意为解决方案付费。

执行步骤：

1. 先从微信现有关系链里找 20 到 30 个最接近目标用户的人。
2. 按身份分层，至少覆盖 3 类用户，避免只听单一圈层反馈。
3. 先让他们试听或试用，再收集“是否真有帮助”和“愿意付多少钱”两类反馈。
4. 必须以真实付款作为验证标准，不能只收口头认可。
5. 根据付费和反馈结果，决定前端低价验证品和后端高客单承接品的分层。

案例来源：

- Luke 林案例中的 25 人试听、1200 元收款、9.9 元课与千元社群分层

### AP003 - 单入口私域承接改造 SOP

适用目标：内容已能带来加微或进群，但微信侧承接和交付开始卡顿。

执行步骤：

1. 先把当前路径写成单线：看到内容 -> 点击入口 -> 领资料/付款 -> 进入交付。
2. 把微信欢迎语改成“一句定位 + 一句承诺 + 一个入口”。
3. 把资料领取、低价课支付和课程入口迁到同一个系统入口。
4. 把高客单申请入口放到前端交付完成后的下一屏，而不是放在私聊里解释。
5. 迁移后只观察 24 小时：资料追问次数、支付转化率、产品差异追问次数。

案例来源：

- Luke 林案例中的小鹅通迁移
- 阶段二反馈会话中的自修正工单

### TR001 - 结果型标题模板

可直接套用：

1. `我是[身份]，用[打法]在[时间]做到[结果]`
2. `[人群]如何在[限定条件]下完成[结果]`
3. `从[糟糕起点]到[可验证结果]，我做对了什么`

### TR002 - 收费验证记录模板

记录字段：

- 用户类型
- 原始痛点
- 是否试听/试用
- 反馈摘要
- 愿意付费价格
- 是否真实付款
- 是否适合进入后端承接

### TR003 - 产品边界说明卡模板

固定写法：

- `9.9 元课适合谁`
- `9.9 元课交付什么`
- `千元社群适合谁`
- `千元社群交付什么`
- `什么时候不建议升级到社群`

### TC001 - B站冷启动发布检查清单

发布前必须检查：

1. 标题是否明确了结果和人群。
2. 开头 30 秒是否先讲身份和结果。
3. 全文或视频是否只有一个 CTA。
4. CTA 是否落到单一入口，而不是多入口分流。

### TC002 - 微信收费验证执行指令

执行要求：

1. 邀约文案只说明目标和时长，不先讲一堆产品理念。
2. 试听结束后必须收“是否愿意付费”和“愿意付多少”的反馈。
3. 真正统计时只认付款，不认客气话。

### TC003 - 统一入口配置清单

配置要求：

1. 统一一个资料入口。
2. 统一一个支付入口。
3. 统一一个课程或交付入口。
4. 微信欢迎语只指向这个入口。
5. 高客单说明不放欢迎语第一屏。

## 连接边

### E001

- `(U001) --【优先追求】--> (G001)`
- 触发条件：用户为个人副业者，启动资金小于 2000 元，日均可用时间小于 3 小时。
- 调用产出：进入低成本、低配置的获客路径，默认组合 `AP001 + TR001 + TC001`。
- 不适用警报：如果用户已有稳定新增且问题发生在支付或交付环节，则不应继续停留在“获取初始精准流量”，而应切到承接问题。

### E002

- `(G001) --【首选策略】--> (N001)`
- 触发条件：用户需要先完成公域破冰，且有真实身份、真实经历或真实案例可讲。
- 调用产出：激活 `N001` 时，自动组合 `AP001 + TR001 + TC001`。
- 不适用警报：如果没有真实结果或没有可引用案例，不应硬上强结果承诺型内容。

### E003

- `(N001) --【带来第一层验证】--> (N002)`
- 触发条件：内容已经带来评论、私信、关注、加微或群内咨询，但尚未证明用户愿意付费。
- 调用产出：激活 `N002` 时，自动组合 `AP002 + TR002 + TC002`。
- 不适用警报：如果用户没有任何可交付雏形，不能直接收费验证，应先补最低可交付方案。

### E004

- `(N002) --【放大前必须接入】--> (N003)`
- 触发条件：已经出现集中新增、资料领取需求、付款行为或高客单咨询，且人工承接耗时开始上升。
- 调用产出：激活 `N003` 时，自动组合 `AP003 + TR003 + TC003`。
- 不适用警报：如果每天新增极低、还没有真实付款，不要过早上复杂系统，只保留最轻入口。

## 沙盘推演

### 输入用户

- 情境：`U001 个人副业者`
- 目标：`G001 获取初始精准流量`
- 预算：1000 元
- 日均可用时间：2 小时
- 当前资源：有微信好友、有副业经验课雏形、没有成熟公域流量
- 当前限制：不投流、不做重系统、不想一开始就搞复杂工具

### 推演路径

`U001 -> G001 -> N001 -> N002 -> N003`

### 为该用户动态组装的作战方案包

**第一阶段：公域破冰**

- 首选策略：`N001 强结果内容破冰`
- 方案包：
  - 执行 SOP：`AP001`
  - 模板库：`TR001`
  - 工具检查：`TC001`
  - 参考案例：
    - [Luke 林案例](/Users/naipan/.hermes/skills/strategy-material-engine/assets/cases/在B站教年轻人赚钱月入三万，从0成为数字游民，全流程底层逻辑分享.md)

执行目标：

1. 用一个真实结果型主题完成第一次内容破冰。
2. 不追求泛播放量，优先让目标人群产生关注、评论或加微动作。
3. 所有内容只保留一个 CTA。

**第二阶段：付费验证**

- 次级策略：`N002 种子用户收费验证`
- 触发原因：一旦出现第一批感兴趣用户，就要尽快验证“是否有人愿意付费”，而不是一直靠内容热度自我鼓励。
- 方案包：
  - 执行 SOP：`AP002`
  - 模板库：`TR002`
  - 工具指令：`TC002`
  - 参考案例：
    - [Luke 林案例](/Users/naipan/.hermes/skills/strategy-material-engine/assets/cases/在B站教年轻人赚钱月入三万，从0成为数字游民，全流程底层逻辑分享.md)

执行目标：

1. 从微信现有关系链里找到第一批可收费验证的种子用户。
2. 不以“喜欢”作为验证标准，只以“是否付款”作为验证标准。
3. 拆出前端低价验证品和后端高客单承接品。

**第三阶段：承接收口**

- 放大前策略：`N003 单入口私域承接`
- 触发原因：一旦内容和种子用户验证开始带来新增，微信手工承接会迅速成为瓶颈。
- 方案包：
  - 执行 SOP：`AP003`
  - 模板库：`TR003`
  - 工具指令：`TC003`
  - 参考案例：
    - [Luke 林案例](/Users/naipan/.hermes/skills/strategy-material-engine/assets/cases/在B站教年轻人赚钱月入三万，从0成为数字游民，全流程底层逻辑分享.md)
    - [阶段二承接卡顿诊断会话](/Users/naipan/.hermes/skills/strategy-material-engine/expert_models/sessions/2026-04-21_bilibili_private_conversion_bottleneck.md)

执行目标：

1. 把资料领取、低价课支付和正式交付收成一个入口。
2. 高客单申请不放在私聊临时解释，而是放到前端交付的下一步。
3. 迁移后只看 24 小时数据，不继续做迁移前的长时间人工统计。

### 风险提示

1. 不推荐一开始上投流广告；当前情境下预算和时间都不支持。
2. 不推荐一开始购买重型私域系统；先完成“单入口”比先上复杂工具更关键。
3. 不推荐用没有真实结果支撑的强承诺标题；这会破坏后续成交信任。
