# Buildmate 系统管理员操作手册

这份手册只服务试运行期，目标是让系统管理员**不看规则文档也能正确喂系统**。

补充说明：

- 这份文档主要给系统管理员看
- 如果你是 agent，当前任务又是**阶段一提速执行**，默认先读 `references/buildmate/stage1-fast-path.md`
- 阶段一成功路径真实失败前，不要先回到本手册补读长模板
- 不要为了跑阶段一成功路径，先把完整 workflow / schema / quality-bar 全部吃进上下文

使用原则只有 4 条：

1. 先判断你现在要走的是 **阶段一建库**、**阶段二问诊**、**阶段二反馈**、**阶段三策略推演**，还是 **阶段四智能体闭环**
2. 进入对应阶段后，**直接复制模板**
3. 不要自由发挥字段名，尽量按模板原样填写

补充说明：

- 试运行期间，系统应采用**混合模式进度汇报**
- 也就是：
  - 每 `3` 分钟至少汇报一次
  - 阶段切换 / 关键步骤开始时立即汇报
- 每次汇报至少应包含：
  - 当前阶段
  - 当前步骤
  - 当前动作
  - 预计剩余时间

如果系统在长时间整理 / 召回 / 自修正过程中没有任何进度汇报，可以视为**未遵守试运行协议**

## 一、原贴提交模板

适用场景：

- 你手上是一篇**原始帖子 / 原始复盘 / 原始访谈内容**
- 还**没有**按 `案例身份证 / 决策地图 / 核心心法` 拆过
- 你希望系统把它送进阶段一，做六步拆解和后续入库准备

阶段一第一步固定动作：

- 先在一个简单文档工具里搭“案板”
  - 飞书文档 / Word / Notion / 备忘录都可以
- 文档最顶部先写：
  - `帖子标题`
  - `原文链接`
- 再往下写：
  - `【案例身份证】`
  - `【决策地图】`
  - `【核心心法】`
- 不要一上来先建复杂表格、数据库或多维表
- 可直接复用 `references/buildmate/stage1-workbench-template.md`

阶段一第二步固定动作：

- 打开选定原贴，从头到尾先完整读一遍
- 这一遍只准像读故事一样读，不要做笔记，不要划线，不要边读边填字段
- 读完后，先只用一句大白话回答：
  - 这个人最开始啥情况，后来想干嘛，最后干成了啥
- 回到 `【案例身份证】` 后，第一轮只填写 `一句话业务`
- 其他身份证字段留到第二遍精读再补

阶段一第三步固定动作：

- 进入第二遍精读后，只找四类具体信息：
  - `作者是谁`
  - `启动资源`
  - `核心目标`
  - `最终结果`
- `作者是谁` 重点看作者身份、背景和原先在做什么
- `启动资源` 重点看资金、时间、是否单人、技能、设备
- `核心目标` 重点看他一开始最想达成的具体结果
- `最终结果` 重点看收入、粉丝、周期等关键结果数据
- 数字、周期、量化信息尽量按原文抄，不要改写虚化
- 这一轮结束时，`【案例身份证】` 应该补满

阶段一第四步固定动作：

- 第三遍细读时，专门找作者做“选择”的地方，也就是所有“十字路口”
- 重点留意这些表达：
  - `我决定...`
  - `我选择了...`
  - `在 A 和 B 之间，我选了...`
  - `我试了 X，但不行，于是改成 Y`
  - `最重要的一个动作是...`
- 每找到一个，就在 `【决策地图】` 下新增一个 `决策点`
- 每个 `决策点` 至少要写：
  - `决策点`
  - `选择`
  - `依据`
  - `动作`
- `依据` 必须回到上下文解释他为什么这样选
- `动作` 必须尽量写到工具和操作层，不要只写抽象动作名
- 原文没有写清楚时，必须显式写 `【原文未提供具体动作，推测可能是：...】`

阶段一第五步固定动作：

- 先关上原帖，不要再回看原文
- 只看你已经写好的 `【案例身份证】` 和 `【决策地图】`
- 提炼 `作战三原则 / 最大一个坑 / 最值钱一句忠告`
- `作战三原则` 必须写成可检查的动作纪律，不能写空泛品质词
- `最大一个坑` 必须附带作者验证过的解决方案
- `最值钱一句忠告` 必须能让一个新手立刻改变一个行为

阶段一第六步固定动作：

- 从头到尾再看一遍整份拆解报告
- 检查 `【案例身份证】`、`【决策地图】`、`【核心心法】` 三部分能不能串成一条自洽逻辑
- 把每个 `动作` 项大声读出来，检查新手听完后是否知道第一步做什么
- 如果动作读完还不能直接上手，就回去把动作改写得更具体
- 最后在文档最前面补 `【案例名称】`
- 命名格式固定为 `作者身份 + 核心打法 + 结果`

提交时直接复制：

```text
【输入类型】原贴
【帖子标题】...
【原文链接】...
【来源平台】...
【是否商业化验证】是 / 否 / 不明
【是否明确赚到钱】是 / 否 / 不明
【原文全文或完整摘录】...
【我的备注】...
```

系统收到后，应该进入：

- `阶段一（1/4）输入判别`
- `阶段一（2/4）搭案板并六步拆解`
- `阶段一（3/4）质量校验`
- `阶段一（4/4）注册入库`
- 如果没有缺字段或不可自动修复错误，应继续自动构建索引，不需要你再催一次
- 默认只回结果文件路径、摘要和校验结果；不会直接把整篇拆解稿刷在对话里

不要这样提交：

- 只发一个链接，不贴正文
- 只说“帮我拆一下这个”
- 原贴和自己总结的内容混在一起，但不声明 `【输入类型】`

如果你想直接看完整拆解稿，需要明确说：

- `把完整拆解稿贴出来`
- `不要摘要，直接展开全文`

## 二、病历样提交模板

适用场景：

- 你手上已经有一份**整理好的案例稿**
- 它已经基本具备阶段一的正文结构
- 你要让系统判断：这是**可直接入库**，还是**只能继续训练补全**

提交时直接复制：

```text
【输入类型】整理好的病历样
【是否可直接入库】是 / 否
【案例名称】...
【是否已商业化验证】是 / 否
【是否已赚到钱】是 / 否
【原始来源链接】...
【正文】包含：
## 【案例身份证】
...

## 【决策地图】
...

## 【核心心法】
...

## 【待验证推测】
...
```

系统收到后，应先判断：

- 如果 `【是否可直接入库】是`：优先走 `run_stage1_pipeline.py --input-type case`
- 如果 `【是否可直接入库】否`：先补齐结构，再考虑入库

判断为“可直接入库”的最低标准：

- 有 `案例身份证`
- 有 `决策地图`
- 有 `核心心法`
- 决策点里已经写到动作层
- 原文不确定处已显式写 `【推测】`

不要这样提交：

- 只发一份摘要，没有正文结构
- 说“这是整理稿”，但正文还是原贴
- 已经是病历样，却不写 `【是否可直接入库】`

## 三、工单反馈模板

适用场景：

- 用户已经执行过阶段二工单
- 现在要把执行结果回传给系统
- 希望系统触发 `self_repair`，并把反馈落到对应 `session`

提交时直接复制：

```text
【反馈类型】阶段二执行反馈
【会话文件】expert_models/sessions/xxxx.md
【原始痛点】...
【执行任务】T1, T2
【未执行任务】T3
【结果变化】...
【是否改善】是 / 否 / 部分
【是否赚到钱】是 / 否
【金额】...
【新暴露出的卡点】...
【我建议修正的地方】...
【证据】数据 / 文本 / 截图说明
```

最小可触发字段：

- `【会话文件】`
- `【执行任务】`
- `【结果变化】`
- `【是否改善】`

只要这 4 项齐了，系统就应：

1. 把反馈写入对应会话的 `用户反馈`
2. 触发 `self_repair`
3. 重新判断原工单哪些任务有效、哪些无效
4. 必要时触发 `stage1_replenishment`

如果还补了这些字段：

- `【是否赚到钱】`
- `【金额】`

系统就应额外把这次结果视为：

- **阶段一补录 / 回写候选**

注意：

- “我试了下，感觉一般” 这种反馈**不能直接触发系统修正**
- 必须尽量绑定到**具体会话文件**和**具体任务编号**

## 四、临床实习记录模板

适用场景：

- 你刚做完阶段二的第一版表单和工单
- 现在要进入“临床实习”
- 你要正式记录 3 个测试对象的反馈，而不是口头说“我找朋友试了”

提交或记录时直接复制：

```text
【输入类型】阶段二临床实习
【训练场文件】expert_models/training_grounds/xxxx.md
【问诊表单】expert_models/forms/xxxx.md
【诊断手册】expert_models/manuals/xxxx.md
【测试对象1】...
【测试对象1-三问反馈】...
【测试对象1-执行任务】...
【测试对象1-1到2天结果变化】...
【测试对象1-是否改善】是 / 否 / 部分
【测试对象2】...
【测试对象2-三问反馈】...
【测试对象2-执行任务】...
【测试对象2-1到2天结果变化】...
【测试对象2-是否改善】是 / 否 / 部分
【测试对象3】...
【测试对象3-三问反馈】...
【测试对象3-执行任务】...
【测试对象3-1到2天结果变化】...
【测试对象3-是否改善】是 / 否 / 部分
【有效动作】...
【无效动作】...
【下一轮唯一变量】...
【是否已经赚到钱】是 / 否
【金额】...
```

固定三问必须是：

1. 表单问题看得懂吗？
2. 诊断结论你觉得说到点上了吗？
3. 行动建议你觉得能直接操作吗？

如果测试后已经证明“有效 + 赚到钱”，系统应额外生成：

- `阶段一回写候选`

下一步按阶段一入口注册为新案例。

## 五、阶段三策略推演模板

适用场景：

- 你已经有一个明确目标
- 希望系统基于当前案例库和阶段三图谱组装方案包
- 你要的不是阶段二症状诊断，而是阶段三路径规划

提交时直接复制：

```text
【输入类型】阶段三策略推演
【目标】...
【用户类型】...
【平台】...
【业务领域】...
【资源约束】...
```

系统收到后，应进入：

- `阶段三（1/7）目标收集`
- `阶段三（2/7）情境收敛`
- `阶段三（3/7）图谱召回`
- `阶段三（4/7）路由判断`
- `阶段三（5/7）方案包组装`
- `阶段三（6/7）校验收口`
- `阶段三（7/7）交付执行`

默认交付结果应包括：

- 阶段三会话文件路径
- 证据状态
- 主策略 / 组合策略
- 方案包中的动作包、模板、工具包

不要这样提交：

- 只说“帮我规划一下”，不写目标
- 用抽象大词代替目标，例如“我想提升商业能力”
- 把阶段二症状和阶段三目标混在一起，但不声明这是阶段三推演

## 六、阶段四智能体闭环模板

适用场景：

- 你要初始化商业档案
- 你要同步某个前线的数据看板
- 你要把阶段三执行反馈正式回流
- 你要生成每周战略复盘会

提交时可以直接复制其中一种：

```text
【输入类型】阶段四档案初始化
【老板名】...
【月度现金流】...
【每周可投入时间】...
【核心技能】...
【启动资源】...
【12个月核心目标】...
【风险偏好】...
【主战场】...
```

```text
【输入类型】阶段四执行反馈
【来源阶段三会话】strategy_models/sessions/xxxx.md
【执行任务】T1, T2
【结果变化】...
【是否改善】是 / 否 / 部分
【新暴露问题】...
【是否授权更新看板】是 / 否
【是否授权生成每周复盘】是 / 否
```

默认交付结果应包括：

- 档案 / 看板 / 反馈 / 周复盘文件路径
- 当前主判断
- 学习动作
- 是否建议回流阶段二或阶段一

## 六、管理员日常最短口令

如果只是想快速触发，不想回忆完整规则，可以直接用下面 3 句：

- 阶段一原贴：`用 strategy-material-engine 处理这篇原贴`
- 阶段一病历样：`用 strategy-material-engine 注册这个整理好的病历样`
- 阶段二反馈：`这是阶段二执行反馈，请按 strategy-material-engine 触发自修正`
- 阶段三推演：`用 strategy-material-engine 做阶段三策略推演`
- 阶段四闭环：`按 strategy-material-engine 跑阶段四反馈闭环`

但注意：  
**短口令只负责触发，不负责补字段。**  
触发后，系统仍应继续追问你该填的模板字段。

## 七、管理员自检清单

每次提交前，先快速过一遍：

- 我有没有写 `【输入类型】` 或 `【反馈类型】`
- 我这次到底是在走阶段一、阶段二、阶段二反馈、阶段三，还是阶段四
- 如果是阶段一，我有没有说明是 `原贴` 还是 `整理好的病历样`
- 如果是反馈，我有没有写 `【会话文件】`
- 我有没有尽量把信息填成结构化字段，而不是散乱自然语言

如果以上任一项没满足，系统大概率还得继续追问。
